2. 
Selection sort is O(n^2). For every double in input, it is approximately (inconsistently) 4x slower. 
4 = 2^2, so the running time is O(n^2). The inconsistency tells me that its best case scenario is probably 
less, such as O(n), and its worst case is O(n^2).

Insertion sort is O(n^2). For every double in input, it is approximately 4x slower. 4 = 2^2, 
so the running time is O(n^2). This is pretty consistent throughout, which tells me that its best 
and worst case scenario is probably pretty close to O(n^2). 

Merge sort is O(n logn). For every double in input, it's approximately 2.3x slower. 
This corresponds to O(n log n) on the chart. 

Quick sort is O(n logn). For every double in input, it's approximately 2x slower. 
This corresponds to O(n log n) on the chart. Quick sort has moments where doubling the size makes it run
less than 2x slower, which tells me that its best case run time is probably lower, at O(n), bringing
the average down.
